# 量潮数据云贡献指南

感谢您考虑为量潮数据云做出贡献！以下是帮助您开始的指南。

## 行为准则

请遵守本项目的行为准则。我们致力于提供一个友好、安全和欢迎的环境。

## 如何贡献

### 报告错误

在提交错误报告之前，请检查[问题追踪器](https://github.com/quanttide/quanttide-platform/issues)是否已有类似问题。如果您发现的是未被报告的新问题，请使用以下模板提交issue：

**问题标题**：清晰且描述性的问题标题

**问题描述**：
- 问题的详细描述
- 重现步骤
- 期望行为
- 实际行为
- 截图（如果适用）
- 环境信息（操作系统、Python版本等）

### 提交功能请求

功能请求应通过issue提交，并应包括：
- 功能的详细描述
- 该功能将解决的问题
- 可能的实现方法（如果有的话）

### 贡献代码

1. Fork 本仓库
2. 创建您的特性分支 (`git checkout -b feature/amazing-feature`)
3. 提交您的更改 (`git commit -m 'feat(scope): 添加一些amazing特性'`)
4. 推送到分支 (`git push origin feature/amazing-feature`)
5. 打开一个 Pull Request

## 开发流程

### 设置开发环境

请参考根目录下的 [README.md](./README.md) 中的 "快速开始" 部分来设置开发环境。

### 编码规范

- 遵循项目中已有的代码风格
- 提交信息请遵循 [Conventional Commits](https://www.conventionalcommits.org/) 规范
- 为新功能编写单元测试
- 确保所有测试通过 (`uv run pytest` for provider and python_sdk, `flutter test` for studio)

### 拉取请求流程

1. 保持 PR 聚焦于单一问题
2. 确保描述清晰地解释了问题和解决方案
3. 包含相关的截图或录像（如果是 UI 更改）
4. 等待维护者的审查和反馈
5. 根据反馈进行修改
