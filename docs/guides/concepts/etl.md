# ETL

ETL（Extract, Transform, Load）是数据工程中的核心流程，用于将数据从源系统提取、转换后加载到目标系统。

## 三个阶段

**提取（Extract）**
- 从各种数据源读取数据（数据库、API、日志文件等）
- 处理增量数据和全量数据
- 监控数据提取的完整性和时效性

**转换（Transform）**
- 数据清洗：去除重复、缺失、异常值
- 数据标准化：统一格式、编码、单位
- 数据聚合：按维度汇总计算
- 数据关联：连接多个数据源

**加载（Load）**
- 将转换后的数据写入目标系统
- 支持全量覆盖和增量更新
- 处理加载失败和重试机制

## ETL vs ELT

传统ETL在中间层进行转换，ELT则在目标系统（通常是数据仓库）内进行转换。ELT的优势是利用数据仓库的计算能力，减少数据移动。
