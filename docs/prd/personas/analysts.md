# 数据分析师 - 产品需求视角

## 目标

- 快速获取所需数据进行业务分析
- 理解数据的业务含义和质量状况
- 及时交付数据洞察和报表

## 当前痛点

- **数据黑盒**：不知道有哪些数据可用，不知道数据从哪里来
- **沟通链路长**：需要通过数据工程师才能获取或修改数据
- **数据不信任**：不知道数据质量如何，不敢直接用于决策
- **重复查询**：相似的分析需求需要重复写SQL
- **交付周期长**：从有需求到拿到数据，周期长，影响决策时效

## 对产品的期待

- **数据目录**：一目了然的数据资产目录，包含字段说明、血缘关系
- **自助查询**：无需依赖工程师，自助查询和探索数据
- **数据质量透明**：明确的数据质量指标和刷新时间
- **复用模板**：常用分析场景的模板和最佳实践
- **协作标注**：可以在数据上添加业务注释和使用说明

## 使用场景

1. **新分析任务** → 通过数据目录快速找到相关数据表
2. **数据异常发现** → 查看数据质量指标，追溯数据来源和变更历史
3. **临时报表需求** → 使用模板快速生成报表
4. **跨团队协作** → 分享数据查询结果，添加业务注释
