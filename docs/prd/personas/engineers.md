# 数据工程师 - 产品需求视角

## 目标

- 快速构建和部署数据处理流水线
- 监控和维护数据基础设施的稳定性
- 响应数据分析师的数据需求

## 当前痛点

- **重复配置**：每次新建数据处理任务需要重复配置相似的环境和依赖
- **协作成本高**：数据分析师不知道有哪些数据可用，沟通成本高
- **监控困难**：数据处理任务失败时，难以快速定位问题
- **版本混乱**：数据集和数据处理逻辑缺乏版本管理
- **资源浪费**：测试和开发环境搭建成本高

## 对产品的期待

- **模板化**：提供数据处理任务模板，减少重复配置
- **目录化**：自动生成数据资产目录，让分析师自助发现
- **可观测**：实时监控任务状态、数据质量、性能指标
- **版本管理**：数据集和数据处理逻辑的版本控制和回滚
- **快速迭代**：支持开发、测试、生产环境的快速切换

## 使用场景

1. **业务提出新报表需求** → 快速创建数据处理任务，从源数据到报表的全链路自动化
2. **数据源schema变更** → 自动检测变更并预警，快速调整下游任务
3. **数据质量异常** → 实时检测并阻断，触发告警
4. **新成员入职** → 通过目录和文档快速了解数据资产
