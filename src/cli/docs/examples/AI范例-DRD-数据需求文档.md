# 数据需求文档：GitHub 用户活动面板

## 项目背景

某开源社区研究团队需要对一批 GitHub 用户（约数万级）进行活动分析，了解他们的日常开发行为模式。研究团队已有一份目标用户 ID 列表，但缺少这些用户的每日活动汇总数据。

## 数据来源

数据来源于 GHTorrent 项目的 MySQL dump。GHTorrent 是一个开源学术项目，持续抓取 GitHub 公开事件数据并存储为关系型数据库。研究团队持有该 dump 的完整副本。

## 输出要求

### 交付格式

- CSV 文件，每条记录一行
- 按用户标识 + 日期排序

### 输出字段

| 序号 | 字段 | 含义 | 备注 |
|------|------|------|------|
| 1 | 用户标识 | 去标识化的用户 ID | 脱敏处理，不可追溯到 GitHub 真实用户 |
| 2 | 账户名 | 用户 GitHub 登录名 | |
| 3 | 日期 | 活动观察日期 | YYYY-MM-DD 格式 |
| 4 | Push 次数 | 当天推送代码的次数 | |
| 5 | PR 次数 | 当天发起 Pull Request 的次数 | |
| 6 | Issue 评论数 | 当天在 Issue 中评论的次数 | |
| 7 | PR 评论数 | 当天在 PR 中评论的次数 | |
| 8 | PR 合并数 | 当天合并 Pull Request 的次数 | |
| 9 | Commit 评论数 | 当天在 Commit 中评论的次数 | |
| 10 | Bot 标记 | 是否为机器人账号 | 0=人类，1=机器人 |

## 质量要求

| 指标 | 要求 |
|------|------|
| Bot 识别准确率 | 目标 > 90%（当前基线约 70%） |
| 数据去重 | 同一用户同一天不允许出现重复记录 |
| 数据完整性 | 用户列表中的所有用户均需覆盖，无法匹配的标注原因 |

## 交付方式

- 交付物：CSV 文件 + 数据处理说明（含 Bot 识别方法简述）
- 交付时间：两周内，可分阶段（先出部分样本确认质量，再出全量）
- 格式：CSV，UTF-8 编码
