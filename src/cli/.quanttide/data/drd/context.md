# <化妆品功效检测报告标准化处理项目>

## 1. 业务背景
- **客户是谁，做什么业务**  
  客户是一家关注化妆品功效数据管理的机构（或企业），手头拥有多家第三方检测机构（如 SGS、华测、谱尼等）出具的化妆品功效检测报告。
- **当前面临什么问题**  
  由于不同检测机构的报告格式、指标命名、数据结构各不相同，现有几十份 PDF 报告无法直接合并、查询和对比分析。人工逐份录入效率低、易出错，且无法快速追溯到原始报告的具体位置。

## 2. 数据来源
- **客户能提供什么数据**  
  - 数据形式：PDF 格式的检测报告，约几十份。  
  - 数据内容：包含功效检测指标（如皮肤水分含量、TEWL 值、紧致度、弹性等），每份报告含多个样品、多个时间点的检测值（均值、标准差、变化率、P 值等）。  
  - 数据规模：几十份报告（具体份数待确认），每个报告数据量不大但格式差异大。  
  - 更新频率：未明确说明（后续确认是否定期新增报告）。  
- **是否有样例数据**  
  客户可提供部分报告样例供参考（待确认）。

## 3. 期望产出
- **客户希望最终拿到什么**  
  - 一个结构化数据集（Excel 或 CSV 格式），采用**长表格式**——每一行代表“某报告、某样品、某指标、某时间点、某组别”的一次观测值。  
  - 同时，不同机构对同一指标的叫法需**统一映射**为标准指标名（例如“皮肤水分含量”对应 SGS 的“角质层水分含量”、华测的“皮肤含水量”）。  
  - 每个抽取的数值需**附带溯源信息**——可定位到原始报告的具体页面和位置，方便业务方复核。  
- **对产出格式有什么偏好**  
  - 要求保留原始数值精度（如 30.37% 必须保留为 30.37%，不可四舍五入）。  
  - 最终交付物需包括映射字典（原始指标名→标准指标名）和溯源字段。

## 4. 约束与要求
- **时间要求**  
  未明确（后续确认）。  
- **安全/合规要求**  
  原始报告可能包含机构或产品信息，需确保数据处理过程中不泄露敏感内容，仅用于内部标准化。  
- **其他特殊要求**  
  - 数值精度不可丢失。  
  - 所有提取结果必须可追溯至报告原文的具体位置（页码、段落、表格行号等）。  
  - 如果业务方对某数值有疑问，应能直接打开原报告核对。

## 5. 待确认事项
- **具体指标列表**：客户期望的标准指标名称集合是什么？是否已有预定义标准？  
- **报告数量与更新频率**：目前几十份报告是否一次交付，后续是否有新增报告需定期处理？  
- **样品与组别信息**：每份报告中样品数量、组别（如“对照”“实验”）的命名规则？  
- **数值处理规则**：对于缺失值、异常值（如“未检出”）是否特殊处理？  
- **映射规则来源**：不同机构的指标名与标准名之间的映射是由客户提供，还是需要自动化推理后由客户确认？  
- **交付时间与阶段**：是否有分阶段里程碑（如先处理 5 份测试报告验证可行性）？  
- **数据安全要求**：是否需要签署保密协议或进行脱敏处理？
